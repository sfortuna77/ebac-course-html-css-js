Princípios SOLID

Princípio da Responsabilidade Única SRP

O Single Responsibility Principle é o primeiro dos princípios SOLID e estabelece que uma classe deve ter apenas uma razão para mudar.

O que significa?

Cada classe deve ser responsável por apenas uma funcionalidade ou tarefa específica. Isso torna o código mais organizado, fácil de entender e manter.

Princípio Aberto/Fechado OCP

O Open/Closed Principle determina que as classes devem estar abertas para extensão, mas fechadas para modificação.

O que significa?

Você deve poder adicionar novas funcionalidades sem alterar o código existente. Isso é alcançado através de abstrações como interfaces e herança.

Princípio da Substituição de Liskov LSP

O Liskov Substitution Principle estabelece que objetos de uma classe derivada devem poder substituir objetos da classe base sem quebrar a funcionalidade.

O que significa?

Se você tem uma classe filha, ela deve funcionar perfeitamente no lugar da classe pai, mantendo o comportamento esperado.

Princípio da Segregação de Interfaces ISP

O Interface Segregation Principle afirma que uma classe não deve ser forçada a implementar interfaces que não utiliza.

O que significa?

É melhor ter várias interfaces pequenas e específicas do que uma interface grande com muitos métodos desnecessários.

Princípio da Inversão de Dependência DIP

O Dependency Inversion Principle estabelece que módulos de alto nível não devem depender de módulos de baixo nível. Ambos devem depender de abstrações.

O que significa?

Em vez de depender de implementações concretas, suas classes devem depender de interfaces ou abstrações.
